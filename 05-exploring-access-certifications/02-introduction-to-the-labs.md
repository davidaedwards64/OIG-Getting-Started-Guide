## Introduction to the Labs


This section provides an overview of the labs and the prerequisites to
run them.

### The Labs in this Section

This section has guided labs to explore different aspects of Access
Certification:

- **Lab 3.1 - User Access Review** - running user-centric campaigns,
  often referred to as User Access Reviews is common and we will use
  this to show the details of creating, launching and running a
  campaign.

- **Lab 3.2 - Resource Campaign with Entitlements** - the other type of
  campaign is a resource campaign which focuses on specific resources.
  We will show how entitlements can be reviewed for an
  entitlement-enabled campaign.

The lab sections will look at defining/importing entitlements and
defining policy. The access requests and access certifications using
entitlements are covered later in the document.

### Prerequisites

The prerequisites for this section were covered in the earlier
[<u>Preparing Your Okta
Environment</u>](#preparing-your-okta-environment) section:

- OIG enabled in your Okta org

- Users in Okta who are in a group and have applications assigned

- Applications with entitlements and users assigned as we’ve used in the
  other labs in this guide

All three of the apps used earlier (Building Access, Carpark Access and
Salesforce) will be used in the examples, but you could use any apps in
your environment.

Also, if you don’t have an entitlement-enabled application in your
environment, you should run a resource campaign against any app you have
with users assigned (the example will show how to configure it for
entitlements).
